# 水库水质智能监测分析平台 · 开发模块与接口设计

## 1. 文档范围

本文用于指导开发落地，只覆盖：

- 开发模块顺序
- 各模块功能边界
- 页面与后端能力对应关系
- REST 接口设计
- WebSocket / SSE 接口边界
- 核心字段设计
- 模块验收标准

不包含业务背景、宣传描述、长段代码、部署步骤。

| 顺序 | 模块 | 优先级 | 开发目标 |

## 3. 技术栈约定

| 层级 | 技术 | 用途 |
|---|---|---|
| 前端 | Vue 3、Vite、Element Plus、ECharts | 后台页面、表单、图表、图谱展示 |

## 4. 通用接口规范

### 4.1 基础路径

| 类型 | 约定 |
|---|---|
| REST API | /api/v1 |
| WebSocket | /ws |
| SSE | /sse |
| 文件访问 | /api/v1/files |

### 4.2 认证方式

| 项目 | 说明 |
|---|---|
| 登录方式 | 用户名 + 密码 |
| 鉴权方式 | Bearer Token |
| Token 有效期 | 24 小时 |
| 刷新方式 | refresh token 或无感刷新接口 |
| 权限控制 | RBAC，后端校验接口权限，前端控制菜单显示 |

### 4.3 统一返回字段

| 字段 | 类型 | 说明 |
|---|---|---|
| code | int | 业务状态码，0 表示成功 |
| message | string | 提示信息 |
| data | object / array | 返回数据 |
| trace_id | string | 请求追踪 ID，便于排查问题 |

### 4.4 分页字段

| 字段 | 类型 | 说明 |
|---|---|---|
| page | int | 当前页，从 1 开始 |
| page_size | int | 每页数量 |
| total | int | 总记录数 |
| items | array | 当前页数据 |

### 4.5 通用状态码

| 状态 | 说明 |
|---|---|
| 200 | 请求成功 |
| 400 | 请求参数错误 |
| 401 | 未登录或 Token 失效 |
| 403 | 无权限 |
| 404 | 资源不存在 |
| 409 | 数据冲突，如编码重复、预警重复 |
| 422 | 数据校验失败 |
| 500 | 服务端异常 |

## 5. 模块一：基础工程与认证权限

### 5.1 功能描述

| 功能 | 描述 |
|---|---|
| 登录 | 用户输入账号密码，后端校验后返回访问令牌、用户信息、角色权限 |
| 获取当前用户 | 前端刷新页面后通过 Token 获取当前登录用户 |
| 刷新 Token | Token 快过期时刷新访问令牌 |
| 退出登录 | 前端清理 Token，后端可加入 Token 黑名单 |
| RBAC 权限 | 按角色控制接口访问和前端菜单显示 |
| 路由守卫 | 未登录禁止访问业务页面，无权限隐藏菜单或阻止访问 |

### 5.2 接口设计

| 方法 | 路径 | 功能 | 入参字段 | 返回字段 |
|---|---|---|---|---|
| POST | /api/v1/auth/login | 登录 | username、password | access_token、refresh_token、user、permissions |
| POST | /api/v1/auth/refresh | 刷新 Token | refresh_token | access_token、expire_at |
| GET | /api/v1/auth/me | 当前用户信息 | 无 | id、username、real_name、role、permissions |
| POST | /api/v1/auth/logout | 退出登录 | 无 | success |

### 5.3 验收标准

- 未登录访问业务接口返回 401。
- 无权限访问接口返回 403。
- 登录成功后前端可渲染对应角色菜单。
- Token 过期后前端可自动刷新或跳回登录页。

## 7. 模块三：监测数据接入与治理

### 7.1 功能描述

| 功能 | 描述 |
|---|---|
| 自动站数据接入 | 接收自动监测站 RTU 或模拟接口上报数据 |
| 人工采样录入 | 支持监测员录入人工采样结果 |
| 批量导入 | 支持 Excel 导入历史监测数据 |
| 数据校验 | 执行量程检查、变化率检查、跨站一致性检查 |
| 数据入库 | 合格数据写入 MySQL monitoring_record |
| 实时缓存 | 最新数据写入 Redis，并设置 TTL |
| 数据质量标记 | 正常、可疑、无效数据分级存储 |
| 数据查询 | 支持按水库、站点、指标、时间范围查询 |

### 7.2 接口设计

| 方法 | 路径 | 功能 | 入参字段 | 返回字段 |
|---|---|---|---|---|
| POST | /api/v1/monitoring/records | 新增单条监测数据 | reservoir_id、station_id、indicator_id、value、data_source、record_time | record_id、quality_flag、validation_result |
| POST | /api/v1/monitoring/records/batch | 批量写入监测数据 | records | success_count、failed_count、failed_items |
| POST | /api/v1/monitoring/import | 导入 Excel 数据 | file、data_source | task_id、status |
| GET | /api/v1/monitoring/records | 监测数据列表 | reservoir_id、station_id、indicator_id、start_time、end_time、quality_flag、page、page_size | 监测数据分页列表 |
| GET | /api/v1/monitoring/latest | 最新监测数据 | reservoir_id、station_id、indicator_codes | 最新数据列表 |
| GET | /api/v1/monitoring/trend | 指标趋势 | reservoir_id、indicator_id、start_time、end_time、interval | 时间序列数据 |
| GET | /api/v1/monitoring/compare | 多库对比 | reservoir_ids、indicator_id、start_time、end_time、aggregation | 对比数据 |
| POST | /api/v1/monitoring/validate | 数据校验 | reservoir_id、station_id、indicator_id、value、record_time | quality_flag、reason、suggestion |

### 7.3 数据校验规则

| 校验项 | 处理方式 |
|---|---|
| 量程检查 | 超出指标合理范围标记为可疑或无效 |
| 变化率检查 | 与上一条数据差值过大时标记为可疑 |
| 跨站一致性检查 | 同流域相邻站点趋势明显冲突时标记为可疑 |
| 离线检查 | 站点超过阈值未上报，触发设备离线预警 |
| 重复数据检查 | 同一站点、指标、时间点重复上报时拒绝或覆盖 |

### 7.4 验收标准

- 新数据写入后，MySQL 可查到历史记录。
- 最新数据可从 Redis 快速读取。
- 可疑数据不参与核心预警计算，除非人工确认有效。
- 监测记录表必须支持按 reservoir_id + record_time 高效查询。

- 预警详情可关联 RAG 处置建议和图谱溯源结果。

### 10.3 问答接口

| 方法 | 路径 | 功能 | 入参字段 | 返回字段 |

### 10.4 验收标准

- 标准限值类问题优先使用 indicator 表数据校验，避免 PDF 表格解析误差。
