# 水库水质智慧监测平台 · 学习路线指南

> 按「基础支撑 → 数据底座 → 核心业务 → 高阶智能」四阶段递进学习。
> 每个模块遵循 **models → schemas → services → routers → 前端页面** 的认知闭环：先理解数据模型，再看请求响应 Schema，接着是业务逻辑层，然后看路由注册，最后看前端如何消费接口。

## 附：各阶段学习要点

### 第一阶段要点
- 关注 `.env` 配置文件理解各组件连接参数
- 理解 JWT 的 Bearer Token 流程和 `get_current_user` 依赖注入
- 理解 RBAC 装饰器的角色判断逻辑

### 第二阶段要点
- 理解 `monitoring_record` 表如何通过 `reservoir_id`/`station_id`/`indicator_id` 关联三张维度表
- 理解 Redis 缓存的双写策略（写入MySQL后同步缓存）和降级逻辑（缓存不可用回退MySQL）

### 第三阶段要点
- 理解 Collector Pipeline 的 **事务优先** 原则：数据库事务提交后才进行缓存/推送/异步任务
- 理解预警状态机流转：`0待确认 → 1处置中 → 2已解决 → 3已归档`
- 理解 `source=0`（规则判定）和 `source=1`（AI趋势分析）两种预警来源的区别
- 理解 `llm_suggestion()` 的幂等保护（`suggestion_status == 1` 时跳过）

### 第四阶段要点
- 理解 Analyst ReAct Agent 的**双阶段设计**：工具探索阶段 LLM 自由选工具，最终输出阶段用 JsonOutputParser 强制结构化
- 理解 M4 智能问答 Agent 与 M6 Analyst Agent 的区别：前者用户提问驱动、后者定时任务驱动
- 理解 RAG 检索的**双路 RRF 融合**策略（向量检索 + BM25 关键词检索）
- 理解 Report Generator 如何聚合巡检日志/趋势分析/预警数据生成结构化报告
